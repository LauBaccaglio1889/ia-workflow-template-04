# STORY-006: Integración del Agente Conversacional y Endpoints de Chat E2E

## Description

As a user, I want to talk to the AI agent via a dedicated chat endpoint, so that my natural language questions are processed, correct SQL queries are executed, and contextually coherent answers are generated.

## Acceptance Criteria

- [ ] Given a POST request to `/api/v1/sessions/{id}/chat` with a user question, when sent, then it retrieves the last 10-15 messages of conversational history from SQLite and feeds them into the model execution context.
- [ ] Given the Pydantic AI agent, when executing, then it has access to the Postgres tools (`list_tables`, `get_table_metadata`, `execute_query`) and uses them dynamically according to its system prompt.
- [ ] Given a user request, when the agent successfully responds, then the user's question and the agent's final text answer are saved to the `messages` table in SQLite under the respective session ID.
- [ ] Given a tool execution error (e.g. invalid SQL generated by LLM), when caught, then the agent receives the error description in its tool loop so that it can try to auto-correct or output a clean explanation.

## Technical Notes

- Define the Pydantic AI agent in `backend/app/services/agent.py`.
- Define system prompts that explicitly restrict queries to read-only analytical requests.
- Implement the `/sessions/{id}/chat` route in `backend/app/routers/chat.py`.
- Ensure appropriate usage of `async` / `await` for I/O bound database and LLM calls.
